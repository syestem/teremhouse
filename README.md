# ТеремХаус

Одностраничный статический лендинг для премиального загородного коттеджа «ТеремХаус» в Бердске.

Проект сделан без сборщиков, фреймворков и npm-зависимостей. Можно открыть `index.html` двойным кликом или разместить репозиторий на GitHub Pages.

## Структура

- `index.html` — семантическая разметка, SEO-мета, Open Graph, Twitter card, JSON-LD.
- `assets/css/styles.css` — дизайн-система, адаптив, состояния, lightbox, формы.
- `assets/js/main.js` — константы контактов/условий, мобильное меню, FAQ, lightbox, отзывы, форма бронирования.
- `assets/img/` — SVG-плейсхолдеры изображений.

## Что заменить перед публикацией

В `assets/js/main.js` в начале файла замените TODO-константы:

```js
const SITE_CONFIG = {
  whatsappPhone: "70000000000",
  telegramUsername: "teremhouse_todo",
  instagramUrl: "#",
  phoneDisplay: "[TODO: телефон]",
  phoneHref: "tel:+70000000000",
  email: "hello@example.com",
  emailDisplay: "[TODO: email]"
};
```

Там же легко меняются условия бронирования:

- цена за сутки;
- задаток;
- депозит;
- клининг;
- время заезда и выезда;
- способы оплаты.

В `index.html` замените:

- слоты рейтингов и количества отзывов;
- реальные тексты отзывов;
- спорные FAQ-ответы с пометкой `TODO`;
- реквизиты ИП;
- ссылки на правила проживания и политику конфиденциальности;
- iframe карты в блоке «Расположение».

## Как заменить фотографии

Положите реальные изображения в `assets/img/` и сохраните понятные имена:

- `hero`
- `exterior`
- `living`
- `bedroom-1` ... `bedroom-4`
- `spa-pool`
- `sauna`
- `hammam`
- `terrace-1`
- `terrace-2`
- `gallery-1` ... `gallery-4`
- `og-image`

Можно использовать `.jpg`, `.webp` или `.png`. После замены расширений обновите пути в `index.html`.

Рекомендации:

- hero/OG: горизонтальное изображение 1600×900 или больше;
- галерея: WebP или JPG, ширина 1200–1800 px;
- не удаляйте `alt` у изображений, лучше заменить их на описания реальных фото.

## Форма бронирования

Бэкенда нет. Форма собирает данные гостя в готовый текст и открывает:

- WhatsApp через `wa.me`;
- Telegram через `t.me`;
- email через `mailto`.

Кнопка «Отправить» по умолчанию открывает WhatsApp.

## Деплой на GitHub Pages

1. Загрузите файлы в репозиторий GitHub.
2. Откройте `Settings` → `Pages`.
3. В `Build and deployment` выберите `Deploy from a branch`.
4. Укажите ветку `main` и папку `/root`.
5. Сохраните настройки.

Через несколько минут сайт будет доступен по адресу GitHub Pages.

## Проверка

Локально достаточно открыть `index.html` в браузере. Для проверки ссылок мессенджеров сначала замените TODO-контакты в `assets/js/main.js`.
